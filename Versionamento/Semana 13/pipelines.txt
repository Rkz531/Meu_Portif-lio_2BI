1. O que é uma pipeline em um processo de desenvolvimento de software?

Uma pipeline é uma sequência de etapas automatizadas que verifica, testa, constrói e prepara um sistema para ser entregue ou publicado. Ela permite identificar erros mais rapidamente e reduz a necessidade de realizar cada etapa manualmente.

2. Qual é a diferença entre polling e webhook?

No polling, o sistema consulta repetidamente outro sistema para verificar se existe alguma novidade ou atualização. Já o webhook envia uma notificação automaticamente assim que determinado evento acontece, permitindo uma resposta mais rápida.

3. Quais são as três partes principais de um webhook?

As três partes principais são: o evento, que é o acontecimento que dispara o webhook; o endpoint, que é o endereço para onde os dados são enviados; e o payload, que contém as informações enviadas, normalmente no formato JSON.

4. Por que é importante usar um secret e uma assinatura HMAC em webhooks?

O secret e a assinatura HMAC são importantes para verificar se a mensagem realmente foi enviada por uma fonte confiável e se o seu conteúdo não foi alterado durante o envio. Isso aumenta a segurança, garantindo a autenticidade e a integridade dos dados.

5. Por que uma mensagem como “ERRO! Build quebrado” é menos útil do que uma mensagem que informa serviço, autor, commit e link dos logs?

A mensagem “ERRO! Build quebrado” apenas informa que existe um problema, mas não apresenta detalhes suficientes para resolvê-lo. Uma mensagem que informa o serviço afetado, o autor, o commit e o link dos logs ajuda a equipe a localizar a causa do erro e decidir rapidamente qual deve ser o próximo passo.
